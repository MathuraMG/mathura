In collabaration with Claire Kearney-Volpe and Cassie Tarakajian Worked as the accessible developer on the project

Coding is a means of production and self-expression that should be accessible to everyone. This project is an attempt to level the playing field with people that have low vision or blindness and want to learn to code. In line with The Processing Foundations mission to promote software literacy and empower people of all interests and backgrounds to learn how to program and make creative work with code, the Processing Accessibility Project is a community-based research and development effort involving a large stakeholder network from academia, target end-userâ€™s interested in learning how to code and Processings open-source community.
Claire and myself worked on making the p5 IDE accessible. While the project is still in its early stages here are the main features.

The Layout

The website itself is highly accessible with all elements being accessible by screen readers ( tested with JAWS/IE, OSX/Safari, NVDA/Firefox). This includes the editor as well.

Accessible Options

Apart from the standard web elements, this IDE contains elements that are inclusive of a pull programming experience. Two important features included audio warning for lint messages (which are also visually indicated) and a text output for visual components in the output canvas.
